## v1 Requirements — Final Status

### Data Integrity

- [x] **DATA-01**: User can reimport statements and card bills for an already-imported period without duplicating transactions or deleting data from other sources.
  - **Outcome:** Validated. Deduplicação SHA-256 por arquivo; reimport de qualquer fonte não apaga dados de outras fontes. Testado com 3 PDFs Nubank reais.

- [x] **DATA-02**: User can export the full local dashboard dataset to a portable JSON backup file.
  - **Outcome:** Validated. `full-backup-io.js` exporta todos os stores IndexedDB em JSON único. Backup por store individual também disponível via `config-io.js`.

- [x] **DATA-03**: User can restore a full dashboard backup from a JSON file into local storage with validation of imported data.
  - **Outcome:** Validated. Restore com validação de schema; aceita backups de versões anteriores que não tinham stores de categorização (backward compat).

### Analytics

- [x] **ANLY-01**: User can view spending trends by category across multiple months using imported financial data.
  - **Outcome:** Validated. Trend chart multi-mês por categoria em `analytics.js`; visível no painel de analytics dentro da aba Lançamentos.

- [x] **ANLY-02**: User can compare month-over-month category movement in a way that highlights where spending increased or decreased.
  - **Outcome:** Validated. Movers panel com delta R$ e % por categoria vs. mês anterior; cores diferenciadas para alta/queda.

### Projection

- [x] **PROJ-01**: User can have Registrato/SCR commitments automatically included in projection calculations when compatible imported data exists.
  - **Outcome:** Validated. `projection-model.js` consome snapshots SCR diretamente; commitments aparecem como saídas mensais na Projeção sem entrada manual.

- [x] **PROJ-02**: User can understand which projection values are being influenced by Registrato/SCR-derived commitments.
  - **Outcome:** Validated. Status visual `Incluído` / `Conflito` / `Contextual` diferencia origem de cada linha; painel SCR dedicado com resumo antes da tabela.

### Categorization

- [x] **CATG-01**: User can create, edit, and remove custom categorization rules for imported transactions.
  - **Outcome:** Validated. Interface de gerenciamento inline na aba Lançamentos (painel de categorização); regras persistidas em `categorizacao_regras` store.

- [x] **CATG-02**: User-defined categorization rules are applied to future imports before fallback default categorization rules.
  - **Outcome:** Validated. `buildCategorizationRuntime()` aplica regras do usuário com precedência; categorizador padrão usado apenas como fallback.

- [x] **CATG-03**: User can have a manual category correction remembered and reused automatically on future matching transactions.
  - **Outcome:** Validated. Correção manual gera `buildCategoryMemoryRecord()`; memória persistida em `categorizacao_memoria` store e reaplicada em imports seguintes e recategorização retroativa.
